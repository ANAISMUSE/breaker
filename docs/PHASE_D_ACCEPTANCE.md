# Phase D 验收清单（可打勾）

范围：`应用画像` + `平台对比全量图表`。

## 必要项（上线前必须通过）

- [x] 应用画像构建：前端可提交 `rows`，后端调用 `/api/persona/build` 返回画像结果
- [x] 数据导入：应用画像与平台对比页均支持 `rows` 文件导入
- [x] 平台对比计算：前端可提交 `rows + benchmark + rounds`，后端 `/api/simulation/compare` 正常返回
- [x] 趋势图：展示多策略茧房指数随轮次变化曲线（含中文策略名）
- [x] 末轮对比图：展示“静态基线 vs 模拟末轮”柱状图
- [x] 改善排名图：展示不同策略 `drop`（Δ）横向排序
- [x] 最优策略提示：页面展示 `_best` 结果（策略名/静态指数/改善量）
- [x] 原始结果可追溯：前端可查看原始响应 JSON（便于调试和评审）
- [x] 基础异常处理：输入 JSON 非法或接口失败时前端有错误提示

## 可选增强项（建议在 Phase D+ / Phase E 前置）

- [ ] 对比结果导出（JSON/CSV）与图表导出（PNG/SVG）
- [ ] 固定数据集回放（评审复现实验，含 rounds/benchmark 快速模板）
- [ ] 图表交互增强（策略显隐、区间缩放、hover 联动）
- [ ] 平台对比历史记录页（读取 `/api/simulation/records`，支持筛选与重放）
- [ ] 大样本性能优化（异步计算、进度提示、超时与降级策略）
- [ ] E2E 用例覆盖（应用画像构建、平台对比三图、异常路径）

## 差距闭环（按优先级推进）

### P0（当前迭代必须闭环）

- [x] **结果导出能力缺口**
  - 现状：结果仅可在页面查看，无法形成评审留档
  - 闭环动作：增加“导出结果 JSON/CSV + 导出图表 PNG”按钮
  - 验收口径：同一份运行结果可在 3 次导出中保持内容一致且可读

- [x] **固定回放能力缺口**
  - 现状：评审时依赖手工粘贴数据，难以复现对比结论
  - 闭环动作：内置 1~2 组标准数据集与参数模板，一键填充并运行
  - 验收口径：评审环境可在 1 分钟内复现同一结论与图表

### P1（稳定性与可维护性）

- [x] **历史记录闭环不足**
  - 现状：后端已支持 `/api/simulation/records`（MySQL），前端暂无历史页
  - 闭环动作：新增“历史记录”视图，支持查看、筛选、回放
  - 验收口径：可从历史记录一键回放到当前对比页，并得到一致图形结果

- [x] **自动化回归不足**
  - 现状：Phase D 关键流程缺少前端 E2E 兜底
  - 闭环动作：补齐核心 happy path + 错误路径测试
  - 验收口径：CI 中相关用例稳定通过，关键回归可自动发现

### P2（体验增强）

- [ ] **图表可用性增强**
  - 现状：图表已可展示，但交互能力偏基础
  - 闭环动作：补充图例显隐、缩放、数据点提示优化
  - 验收口径：评审可在不改数据的前提下完成策略横向对比解读

## 通过标准

- 必要项全部勾选为通过，且前后端构建成功；
- P0 差距项全部闭环，P1 至少完成 1 项；
- 应用画像与平台对比可独立完成“导入数据 -> 运行 -> 解释 -> 留档”的最小业务闭环；
- JSON 与 MySQL 两种存储后端行为保持一致（至少保证功能级一致）。
