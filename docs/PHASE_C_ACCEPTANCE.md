# Phase C 验收清单（可打勾）

范围：`人设库 CRUD` + `风险详情页`。

## 必要项（上线前必须通过）

- [x] 人设构建：前端可提交 `rows`，后端生成画像并入库
- [x] 人设列表：可展示 `profile_id/user_id/created_at`
- [x] 人设新建：前端可手动创建人设记录（JSON），后端持久化
- [x] 人设详情：前端可按 `profile_id` 拉取详情并预览画像
- [x] 人设更新：前端可编辑 `user_id/profile`，后端落库
- [x] 人设删除：前端可删除单条记录（含二次确认）
- [x] 风险详情计算：前端可提交 `rows + benchmark`，后端返回详情指标
- [x] 风险详情展示：前端展示概览指标、分布细节、判别量与建议
- [x] 基础异常处理：详情不存在/参数非法时前端有错误提示

## 可选增强项（建议在 Phase D+）

- [ ] 人设记录分页与关键词检索
- [ ] 人设记录导入/导出（单条与批量）
- [ ] 风险详情结果导出（JSON/PDF）
- [ ] 风险详情固定数据集回放（便于评审复现）

## 差距闭环（按优先级推进）

### P0（当前迭代必须闭环）

- [ ] **风险详情结果留档缺口**
  - 现状：风险详情可在线查看，但缺少标准化导出与评审留档
  - 闭环动作：新增风险详情结果导出（至少 JSON，建议补 PDF）
  - 验收口径：同一输入数据 3 次导出结果一致，且可用于评审复核

- [ ] **固定回放能力缺口**
  - 现状：风险详情评审依赖手工粘贴数据，复现成本高
  - 闭环动作：提供固定数据集与参数模板，一键回放风险详情
  - 验收口径：评审环境可在 1 分钟内复现同一指标与结论

### P1（稳定性与可维护性）

- [ ] **人设检索与分页不足**
  - 现状：人设列表仅支持基础展示，数据量上升后检索效率下降
  - 闭环动作：补齐关键词检索 + 分页能力（前后端协同）
  - 验收口径：在中等数据量场景下可稳定完成快速定位与翻页

- [ ] **自动化回归不足**
  - 现状：人设 CRUD 与风险详情链路缺少 E2E 兜底
  - 闭环动作：补齐核心 happy path + 关键异常路径自动化测试
  - 验收口径：CI 中相关用例稳定通过，关键回归可自动发现

### P2（体验增强）

- [ ] **人设导入/导出增强**
  - 现状：已支持人设创建和编辑，但跨环境迁移能力不足
  - 闭环动作：支持单条与批量导入导出（含格式校验）
  - 验收口径：支持最小可用的数据迁移闭环并给出友好错误提示

## 通过标准

- 必要项全部勾选为通过，且前后端构建成功；
- P0 差距项全部闭环，P1 至少完成 1 项；
- 人设 CRUD 与风险详情页可独立完成“录入/编辑 -> 评估 -> 复核/留档”的最小业务闭环；
- JSON 与 MySQL 两种存储后端行为保持一致。
