# Phase B 验收清单（可打勾）

范围：`设备管理` + `任务详情/快照/日志导出完善`。

## 必要项（上线前必须通过）

- [x] 设备列表：可展示 `device_id/name/platform/status/last_seen`
- [x] 设备新增：前端创建设备，后端落库（JSON/MySQL）
- [x] 设备状态更新：前端可切换状态，后端持久化
- [x] 任务列表：可展示 `task_id/name/strategy/status/rounds/created_at`
- [x] 任务详情实时拉取：详情弹窗打开后调用后端详情接口
- [x] 快照查看：任务详情可查看完整 `snapshot` JSON
- [x] 快照追加：任务详情可追加 snapshot（JSON）并即时刷新
- [x] 任务日志记录：创建任务/更新状态/追加快照会生成任务日志
- [x] 任务日志查看：任务详情展示日志（时间/级别/事件/详情）
- [x] 日志导出：前端可导出任务日志 JSON
- [x] 导出任务：前端可导出任务 JSON
- [x] 基础异常处理：任务不存在返回 404，前端显示错误提示

## 可选增强项（建议在 Phase B+ 或 Phase C 前置）

- [x] 设备删除（单条删除，含二次确认）
- [x] 任务日志筛选（级别 + 事件关键字）
- [x] 任务日志分页（大日志量性能优化）
- [ ] 日志导出 CSV/NDJSON
- [ ] 任务详情内“手动写日志”入口（用于运维/实验备注）
- [ ] 快照结构校验（schema 校验与友好错误提示）
- [ ] 操作确认弹窗（危险操作二次确认）
- [ ] E2E 用例覆盖（设备管理、任务详情、日志导出）

## 差距闭环（按优先级推进）

### P0（当前迭代必须闭环）

- [ ] **日志留档能力缺口**
  - 现状：任务日志已支持 JSON 导出，但缺少 CSV/NDJSON 等分析友好格式
  - 闭环动作：补齐日志导出 CSV/NDJSON，并统一字段映射与时间格式
  - 验收口径：同一任务日志可稳定导出多格式，字段完整且可被常见工具读取

- [ ] **快照质量门禁不足**
  - 现状：可追加 snapshot，但对结构有效性缺少强校验
  - 闭环动作：新增 snapshot schema 校验与前端友好错误提示
  - 验收口径：非法快照被明确拒绝且错误可定位，合法快照可正常落库

### P1（稳定性与可维护性）

- [ ] **操作安全兜底不足**
  - 现状：删除等高风险操作的二次确认覆盖不完整
  - 闭环动作：统一危险操作确认弹窗与文案（删除/覆盖/导出覆盖等）
  - 验收口径：关键高风险入口均具备一致确认流程，误操作率可控

- [ ] **自动化回归不足**
  - 现状：设备管理与任务详情关键流程缺少 E2E 兜底
  - 闭环动作：补齐核心 happy path + 错误路径自动化测试
  - 验收口径：CI 中相关用例稳定通过，关键回归可自动发现

### P2（体验增强）

- [ ] **任务备注能力缺口**
  - 现状：任务日志来自系统事件，人工备注入口缺失
  - 闭环动作：在任务详情新增“手动写日志”入口（带级别/标签）
  - 验收口径：运维或实验人员可对任务写入结构化备注并可检索导出

## 通过标准

- 必要项全部勾选为通过，且前后端构建成功；
- P0 差距项全部闭环，P1 至少完成 1 项；
- 新增功能具备最小可用错误提示；
- JSON 与 MySQL 两种存储后端行为保持一致。
